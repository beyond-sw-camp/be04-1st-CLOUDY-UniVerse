5. 자동화 및 프로세스 최적화:

### 시나리오

대학 통합 관리 시스템은 학생이 수강 관리, 출결 관리, 학적 조회, 대여 서비스 관리, 건물의 정보를 관리한다.

1. 학생
    - 학생은 학교에서 발급해준 학번과 PW로 **로그인**이 가능하다.
    - 학생은 학번을 제외한 **개인정보(PW, 주소 등) 수정**이 가능하다.
    - 학생은 PW를 분실하였을 경우, 학생 인증을 통해 **재설정** 가능하다.
    - 학생은 수강 신청 기간에 **수강을 신청**할 수 있다.
    - 학생은 **기자재 대여 / 강의실 대관**이 가능하다.
    - 학생은 본인의 **학적을 조회**할 수 있다.
    - 학생은 수강 신청 기간 이전에 원하는 강의를 **장바구니에 미리 담을** 수 있다.
    - 학생은 수강하고자 하는 **강의를 신청**할 수 있다.(중복)
    - 학생은 학과의 **커리큘럼을 조회**할 수 있다.
    - 학생은 수강 중인 강의 **출석을 조회**할 수 있다.
    - 학생은 자신의 **수강 중인 과목을 조회**할 수 있다.
    - 학생은 **교수의 정보(이력, 교수실, 연구실, 논문 등)를 조회**할 수 있다.
2. 교수
    - 교수는 학교에서 발급해준 ID와 PW로 **로그인**이 가능하다.
    - PW를 분실하였을 경우, **재설정** 가능하다.
    - 교수는 **기자재 대여 / 강의실 대관**이 가능하다.
    - 교수는 자신의 정보(이력, 교수실, 연구실, 논문 등)를 **열람하고 수정**할 수 있다.
    - 교수는 자신의 **강의를 등록**할 수 있다.
    - 교수는 자신의 강의를 수강하는 학생의 **출결을 관리**할 수 있다.
    - 교수는 자신의 강의를 수강하는 학생의 **성적을 결정**할 수 있다.
    - 교수는 **학생의 정보를 조회**할 수 있다.
3. 확장 기능
    - 사용자는 시스템에서 제공하는 **실내 혼잡도 정보를 강의실 별로 조회**할 수 있다.
    - 사용자는 시스템을 통해 **교내 셔틀버스의 실시간 위치를 조회**할 수 있다.
    - 사용자는 시스템을 통해 **멘토링에 참여**할 수 있다.
    - 멘토링에 참여하는 사용자는 해당 **멘토링을 운영하고 관리**할 수 있다.

### 시스템(기능명세)

- 정보 관리
    - 시스템은 학생 정보, 교수 정보, 과목, 강의의 정보를 관리한다.
- 대여, 대관 관리
    - 시스템은 대여, 대관에 관한 기자재, 대여, 대관 정보를 관리한다.
- 수강신청 관리
    - 시스템은 수강신청에서 장바구니를 통해 담긴 강좌를 관리한다.
    - 시스템은 수강신청에서 수강 과목 인원을 관리하여 제한 인원을 초과하지 않도록 한다.
- 출결 관리
    - 시스템은 한 강의에 해당하는 학생들의 출결 정보를 교수에게 제공하고, 학생은 자신의 출결정보만 조회할 수 있다.
- 실내혼잡도 제공(확장)
    - 시스템에서 교내 시설 내 혼잡도 정보를 수신하여 사용자가 특정 장소에 대한 혼잡도 정보를 조회할 수 있다.
- 교내 셔틀 실시간 위치 제공(확장)
    - 시스템에서 교내 셔틀버스에 대한 실시간 위치정보를 수신하여 사용자가 해당 정보를 조회할 수 있다.
- 학생들 멘토링(확장)
    - 방학 기간 중 학생들이 원하는 멘토링 주제와 계획을 학교에 인가 받으면 학교 강의처럼 학기 중 정한 요일, 시간대에 지정된 강의실에서 멘토링을 진행할 수 있다. 이 멘토링은 지도 교수가 지정되며 완료 후
      학점 또는 수료증을 제공한다.

### 개선 내용

---

**강의 테이블 / 과목 테이블**

- 강의 : 과목 = 1 : M 관계 → 강의 : 과목 = M : 1 관계
  강의 테이블의 커리큘럼 학년 컬럼 과목 테이블로 이동
    - 강의 테이블이 과목 테이블에 종속되는 관계가 적합하다고 판단하여 부모-자식 관계 변경
- 학년 제한 삭제
    - 학년 제한은 필요하지 않다고 판단하여 삭제

**학생 테이블**

- 주민등록번호, 성별, 국적, 과정 구분 : null able → not null
    - 학생 정보에 반드시 필요한 정보라고 판단하여 not null 로 변경
- 학적상태 : null able → not null / default : “재학중”
    - 학적 상태 반드시 필요하므로 not null 변경
    - 학생 데이터 입력 시 신입 입학 때 insert가 가장 많이 되므로 기본값을 “재학중” 으로 설정
- 평균학점 → 평균성적
    - 학점이 이수 학점에도 쓰이는 단어라 혼동할 여지가 있어 성적으로 변경
- 비밀번호 추가
    - 학생 로그인 시 비밀번호가 필요하다 판단하여 추가
    - 비밀번호 타입 용량을 크게 둔 이유는 실제 서비스를 할 때 비밀번호 원시값을 넣기 보다는 보안을 위해 해시화 또는 암호화 하여 기입. 따라서 실제 비밀번호보다 길이가 길어지므로 용량을 크게 둠.

**교수테이블**

- 비밀번호 추가
    - 교수 로그인 시 비밀번호가 필요하다 판단하여 추가

**기자재**

- 기자재 상세 위치 삭제
    - 기자재 고장, 등의 이유로 폐기되거나 기자재실 내 대대적인 구조 개편, 기자재실 이동, 등을 할 때 각 값들을 하나하나 변경하기 어려움.
      실제 운용 시에는 담당자가 자신의 규칙의 따라 보관하므로 필요가 없다고 판단하여 삭제

**강의**

- 학년 제한 삭제
    - 권장 학년은 있지만 학년 자체로 신청 자체를 제한하는 경우는 없고, 1학년이 4학년 과목을 신청해도 과사무실에서 권유를 할 뿐 강제로 취소하는 경우는 없다고 판단하여 삭제

## 3. 반정규화 이후 ERD

![반정규화 이후 ERD](https://github.com/beyond-sw-camp/be04-2nd-rename/assets/122339395/32b6134a-49b9-4874-9d6a-377cb63dc65d)

### 반정규화 및 개선 내용

---

**이수강의, 수강신청, 장바구니**

- 학번/강의코드 복합키 → 학번(PK) 강의코드(FK) 로 분리
    - 복합키 사용 시 유지보수가 어렵고, 경우에 따라 성능이 하락하는 경우가 존재
      이수강의, 수강신청, 장바구니 는 학생이 주체가 되는 경우가 많고, 실제 조회도 학생을 기준으로 조회하는 경우가 많을 것이라 판단하여 학번만 PK로 두고 강의코드는 FK로 변경

**출석부**

- 학번/강의코드 복합키 → 강의코드(PK) 학번(FK) 으로 분리
    - 복합키 사용 시 유지보수가 어렵고, 경우에 따라 성능이 하락하는 경우가 존재
      출석부 테이블은 교수님이 해당 강의에 대해 학생들의 출석 여부를 입력하는 경우가 가장 많다고 판단
      강의코드를 기반으로 SELECT와 INSERT 가 자주 일어날 것으로 예상
      따라서 강의코드가 주체가 되어 PK로 두고, 학번은 FK로 변경

**강의**

- 제한학과 컬럼 수강가능학과 테이블로 분리
    - 제한학과의 경우 다수의 과가 제한될 수 있으므로 한 컬럼에서 모두 입력하는 것은 부적절하다고 판단
      학과와 강의의 M:N 관계이므로 수강가능학과 라는 중간 테이블 생성

**강의실**

- 대관 가능 여부 추가
    - 대관 가능 여부로 해당 시설이 수리, 폐쇄 등의 사유로 불가한지, 아니면 빈 대관 시간대에는 신청이 가능한지 보여주기 위해 컬럼 생성

**학생**

- 학생 테이블은 로그인, 출석체크, 대관, 기자재 대여, 수강신청, 등에서 가장 많은 접근을 할 것이라고 판단
  이 경우에는 학번, 이름, 비밀번호 외에는 값을 필요로 하지 않음
  그 외의 경우는 학기 당 한 번, 등과 같이 긴 주기로 CRUD 가 발생할 것으로 판단
  따라서 학적과 관련한 모든 값들은 학적 테이블로 분리해 1:1 관계 생성
  이를 이용해 학생테이블의 값을 요구하는 작업에 대한 성능 향상 기대

### 개선 내용

---

**데이터 타입 기입**

- 각 컬럼에 적합한 데이터 TYPE 지정

    - VARCHAR 의 경우 다음 조건을 따름

  | 컬럼 종류               | 크기          |
              | ----------------------- | ------------- |
  | 이름                    | VARCHAR(50)   |
  | 사유,설명               | VARCHAR(1000) |
  | 이미지                  | VARCHAR(200)  |
  | 주소                    | VARCHAR(300)  |
  | 국적                    | VARCHAR(10)   |
  | 이메일                  | VARCHAR(100)  |
  | 10글자 미만 짧은 문자열 | VARCHAR(30)   |
  | 비번                    | VARCHAR(100)  |
  | 강의평가                | VARCHAR(3000) |

**NULL 여부 모든 컬럼 지정**

- NULL 여부가 지정되지 않은 컬럼에 대해 NULL 여부 지정

**컬럼명 수정**

- 컬럼명에 테이블명이 중복된 경우 테이블명 생략
- 직관적이지 않은 컬럼명의 경우 직관적으로 수정
- 많이 사용되는 약어만 한정하여 약어 사용

**강의실 대관**

- 강의실 대관 테이블과 학생/교수 테이블이 연결되어있지 않아 대관이 불가능 하여 관계 추가
